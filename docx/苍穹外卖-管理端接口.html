苍穹外卖-管理端接口文档

一、员工管理接口

1.1 员工登录

POST /admin/employees/login

请求参数 (JSON)
{
  "userName": "String",    // 用户名
  "password": "String"     // 密码
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "jwt_token字符串"
}

1.2 员工注册

POST /admin/employees/register

请求参数 (JSON)
{
  "userName": "String",    // 用户名
  "name": "String",        // 姓名
  "phone": "String",       // 手机号
  "sex": "String",         // 性别
  "idNumber": "String",    // 身份证号
  "password": "String"     // 密码
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "员工ID"
}

1.3 员工登出

POST /admin/employees/logout

响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "退出成功"
}

1.4 添加员工

POST /admin/employees

请求头

需要携带JWT Token (AdminToken)

请求参数 (JSON)
{
  "userName": "String",    // 用户名 (必填)
  "name": "String",        // 姓名 (必填)
  "phone": "String",       // 手机号 (必填)
  "sex": "String",         // 性别 (必填)
  "idNumber": "String",    // 身份证号 (必填)
  "password": "String"     // 密码 (可选,默认123456)
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "新增员工ID"
}

二、分类管理接口

2.1 添加分类

POST /admin/categories

请求头

需要携带JWT Token (AdminToken)

请求参数 (JSON)
{
  "name": "String",        // 分类名称 (必填)
  "type": "Long",          // 类型: 1菜品分类 2套餐分类 (必填)
  "sort": "Long",          // 排序 (必填)
  "status": "Long"         // 状态: 1正常 2停用 (必填)
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "添加成功::分类ID"
}

2.2 分类分页查询

GET /admin/categories

请求参数 (Query)
name: String      // 分类名称 (可选)
type: Long        // 类型: 1菜品分类 2套餐分类 (可选)
page: Long        // 页码 (必填,默认1)
pageSize: Long    // 每页条数 (必填,默认10)
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "records": [{
      "id": "Long",
      "type": "Long",
      "name": "String",
      "sort": "Long",
      "status": "Long",
      "createTime": "LocalDateTime",
      "updateTime": "LocalDateTime"
    }],
    "total": "Long",
    "size": "Long",
    "current": "Long"
  }
}

2.3 根据类型查询分类

GET /admin/categories/type

请求参数
type: Long    // 类型: 1菜品分类 2套餐分类
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": [{
    "id": "Long",
    "type": "Long",
    "name": "String",
    "sort": "Long",
    "status": "Long"
  }]
}

2.4 更新分类状态

PUT /admin/categories/{id}/status/{status}

路径参数
id: Long      // 分类ID
status: Long  // 状态: 1启用 2禁用
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "更新成功::分类ID::状态"
}

2.5 更新分类信息

PUT /admin/categories/{id}

请求参数 (JSON)
{
  "id": "Long",
  "name": "String",
  "sort": "Long",
  "type": "Long",
  "status": "Long"
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "更新成功::分类ID"
}

2.6 删除分类

DELETE /admin/categories/{id}

路径参数
id: Long    // 分类ID
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "删除成功::分类ID"

七、支付接口

7.1 电脑网站支付

GET /pay/order

请求参数 (Query)
outTradeNo: String      // 商户订单号
subject: String         // 订单标题
totalAmount: String      // 订单金额
productCode: String      // 产品码
响应

跳转到支付宝沙箱收银台页面

7.2 交易查询

GET /pay/order/query

请求参数
outTradeNo: String    // 商户订单号
响应示例
AlipayTradeQueryResponse对象

7.3 退款

POST /pay/refund

请求参数
outTradeNo: String       // 商户订单号
refundAmount: String      // 退款金额
refundReason: String      // 退款原因
响应示例
AlipayTradeRefundResponse对象

7.4 退款查询

GET /pay/refund/query

请求参数
outTradeNo: String       // 商户订单号
outRequestNo: String     // 商户退款单号
响应示例
AlipayTradeFastpayRefundQueryResponse对象

7.5 关闭交易

POST /pay/order/close

请求参数
outTradeNo: String    // 商户订单号
响应示例
AlipayTradeCloseResponse对象

7.6 同步跳转

GET /pay/return

响应示例
已返回商户页面,同步返回。

八、枚举说明

订单状态 (OrderStatus)

值说明
1待付款
2待接单
3已接单
4派送中
5已完成
6已取消
7退款

支付状态 (PayStatusEnum)

值说明
0未支付
1已支付
2退款

支付方式 (payMethod)

值说明
1支付宝
2微信

通用状态

值说明
0禁用/停售/停用
1启用/正常/起售

九、统一响应格式

{
  "code": 200,           // 200表示成功，其他表示失败
  "msg": "操作成功",    // 提示信息
  "data": {}           // 返回数据
}
